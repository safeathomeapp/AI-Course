# AI_COURSE_ROADMAP.md

## Purpose

This document defines the high-level phased roadmap for building the AI-Course project.

It is a planning document, not a detailed task tracker.

It should remain aligned to:

- `COURSE_PHILOSOPHY.md`
- `docs/AI_COURSE_ARCHITECTURE.md`
- `docs/COURSE_CURRICULUM_MAP.md`

---

## Roadmap Overview

The course is built in five phases:

1. define philosophy and architecture
2. develop Tier 1 modules
3. develop Tier 2 modules
4. develop Tier 3 modules
5. produce final course materials

Each phase depends on the stability of the previous phase enough to support the next layer of work.

---

## Phase 1 - Define Philosophy and Architecture

### Purpose

Establish the principles, documentation model, curriculum structure, and development framework that the rest of the course will depend on.

### Expected Outputs

- stable philosophy and AI-usage principles
- repository and documentation architecture
- curriculum map
- module template and scaffold
- review and revision process documents
- initial shared teaching asset strategy

### Dependencies

This is the foundation phase. Later phases should not proceed without enough architectural stability to prevent rework across the whole course.

### Completion at a High Level

Phase 1 is complete when the project has a coherent operating model, a stable documentation system, and enough instructional structure to build modules consistently.

---

## Phase 2 - Develop Tier 1 Modules

### Purpose

Build the Foundations tier that teaches core AI understanding, safe usage, and structured prompting habits.

### Expected Outputs

- drafted and revised Tier 1 modules
- supporting shared assets proven through reuse
- review feedback and synthesis for early modules
- portfolio-oriented learning flow across the foundation tier

### Dependencies

Depends on Phase 1 architecture being stable enough to support consistent module generation, testing, and revision.

### Completion at a High Level

Phase 2 is complete when Tier 1 forms a coherent learning experience that reflects the course philosophy and can serve as the instructional base for later tiers.

---

## Phase 3 - Develop Tier 2 Modules

### Purpose

Extend the course from foundational understanding into practical workflow application in professional contexts.

### Expected Outputs

- Tier 2 module set aligned to the curriculum map
- workflow-oriented examples, exercises, and artefacts
- stronger transfer from module work into practical job tasks

### Dependencies

Depends on Tier 1 establishing the baseline concepts, language, and learner expectations needed for more applied work.

### Completion at a High Level

Phase 3 is complete when learners can move from understanding AI safely to using it in structured professional workflows.

---

## Phase 4 - Develop Tier 3 Modules

### Purpose

Build the systems and architecture tier for learners who need to design, govern, or implement AI-enabled systems.

### Expected Outputs

- Tier 3 module set aligned to the curriculum map
- systems-level teaching material on architecture, reliability, governance, and implementation
- stronger capstone preparation for advanced learners

### Dependencies

Depends on Tier 1 and Tier 2 establishing safe use, workflow thinking, and applied practice before advancing into systems design.

### Completion at a High Level

Phase 4 is complete when the full three-tier curriculum exists as a coherent progression from beginner foundations to system-level thinking.

---

## Phase 5 - Produce Final Course Materials

### Purpose

Convert the full curriculum into a mature, deliverable course system with stable learner-facing materials and capstone support.

### Expected Outputs

- finalized module set
- aligned learner-facing delivery materials
- capstone definitions and supporting materials
- stable cross-tier navigation and portfolio progression

### Dependencies

Depends on the three curriculum tiers being sufficiently complete and internally aligned.

### Completion at a High Level

Phase 5 is complete when the course can be delivered as a coherent end-to-end program rather than a collection of draft instructional assets.

---

## Roadmap Principle

The roadmap should preserve one rule throughout all phases:

build the course as a coherent system, not as disconnected documents or isolated lessons.
