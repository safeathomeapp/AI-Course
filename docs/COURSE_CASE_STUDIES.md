# COURSE_CASE_STUDIES.md

## Purpose

This document defines recurring scenarios used across the course to demonstrate concepts.

Using consistent case studies improves learning because learners see the same types of problems revisited with increasing depth and sophistication.

This file is a reusable reference bank. It should remain aligned to:

- `docs/COURSE_MODULE_TEMPLATE.md`
- `docs/CODEX_MODULE_GENERATION_GUIDE.md`

---

## Usage Guidance

Authors should reuse these recurring domains when they help create continuity across modules.

They are especially useful for:

- demonstrations
- guided exercises
- real-world application sections
- portfolio artefact framing

Reuse a case study when:

- the module benefits from continuity with earlier examples
- the learner gains from seeing the same domain handled with a more advanced method
- the example needs to stay broadly relatable across multiple modules

Use a module-specific example instead when:

- the module topic requires a more precise scenario than the shared bank provides
- a specialized example is necessary to explain the concept clearly
- reuse would make the teaching example feel forced or less clear

## Case Study 1 - Finance and Bookkeeping

Scenario:
A small business must review monthly financial data.

Typical AI tasks:

- summarize financial performance
- detect anomalies
- generate a management report

## Case Study 2 - Software Development

Scenario:
A developer must understand and maintain an unfamiliar codebase.

Typical AI tasks:

- explain code
- generate documentation
- identify potential bugs

## Case Study 3 - Product Management

Scenario:
A product team is deciding whether to launch a new feature.

Typical AI tasks:

- analyze user feedback
- identify risks
- produce a product brief

## Case Study 4 - Marketing and Content

Scenario:
A company wants to launch a new marketing campaign.

## Case Study 5 - Operations and Administration
