# COURSE_MODULE_TEMPLATE.md

## Purpose

This document defines the canonical structure used for every course module.

It is the source of truth for module shape and teaching expectations.

It should remain aligned to:

- `COURSE_PHILOSOPHY.md`
- `docs/COURSE_CURRICULUM_MAP.md`

It is not a generation workflow document. Execution guidance belongs in:

`docs/CODEX_MODULE_GENERATION_GUIDE.md`

---

## General Expectations

- modules should normally take approximately 60-120 minutes
- modules should be written as full teaching lessons, not short outlines
- modules should reinforce responsible AI use, human accountability, and practical professional value

---

## 1. Module Overview

Include:

- module title
- duration
- learning objectives
- prerequisites (optional)

Also include:

- teaching flow and timing, such as concept, demonstration, guided exercise, and reflection blocks

---

## 2. Concept Explanation

Explain:

- why the concept matters
- how the concept works in practice
- common beginner misunderstandings or failure modes

Depth guidance:

- usually 800-1200 words for a 60-90 minute module
- include a practical mental model
- explain why common failure modes happen, such as hallucinations or false confidence

---

## 3. Demonstration

Show:

- example prompt or weak approach
- improved prompt or improved approach
- explanation of why the improvement works

Where possible:

- compare poor and structured approaches clearly
- include realistic examples
- include cross-domain transfer where it helps learners generalize the method

---

## 4. Guided Exercise

Provide:

- task description
- instructions
- suggested prompt structure
- example solution or worked-example support

Also include:

- an evaluation checklist so learners can assess output quality

---

## 5. Real-World Application

Explain how the concept improves real professional workflows.

Include at least one narrative scenario with explicit human review points where appropriate.

---

## 6. Common Mistakes

Include the most relevant errors or weak habits associated with the module topic.

Examples may include:

- vague prompts
- missing context
- weak constraints
- skipped verification

---

## 7. Portfolio Artefact

Define a tangible learner output that can be used beyond the lesson itself.

Examples may include:

- prompt libraries
- workflow diagrams
- structured reports
- decision tools

The artefact should reinforce practical capability, not just theory recall.

---

## 8. Reflection Questions

Include reflection prompts that help learners connect the module to their own work, responsibilities, or decision-making context.

---

## 9. Optional Advanced Exploration

Provide deeper or more demanding extensions for learners who want to go further without overloading the core lesson.
