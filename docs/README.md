# docs/

This folder contains deeper documentation supporting course design, development process, templates, and reusable teaching assets.

Root documents handle project entry points and top-level authority context. The `docs/` folder holds more detailed system, process, template, and reference material.

This structure should remain aligned with:

- `REPOSITORY_MAP.md`
- `docs/AI_COURSE_ARCHITECTURE.md`

## Document Categories

### Authority

- `AI_BEHAVIOUR_AND_USAGE_GUIDELINES.md` - principle-level rules for AI use across the course context
- `AI_COURSE_ARCHITECTURE.md` - course design architecture
- `AI_COURSE_ROADMAP.md` - phased course build roadmap
- `COURSE_CURRICULUM_MAP.md` - canonical module inventory by tier
- `COURSE_MODULE_TEMPLATE.md` - canonical module structure and teaching expectations

### Support

- `MODULE_SCAFFOLDER_TEMPLATE.md` - quick drafting scaffold derived from the canonical module template

### Process

- `CODEX_MODULE_GENERATION_GUIDE.md` - execution workflow for AI-assisted module generation and revision
- `REVIEW_INGESTION_SYSTEM.md` - standard process for converting raw reviews into revision guidance
- `TEST_PANEL_GUIDELINES.md` - framework for collecting structured module feedback

### Reference

- `COURSE_CASE_STUDIES.md` - recurring scenario bank for module examples

## Subdirectories

### `course_assets/`

Reusable teaching assets such as:

- diagrams
- examples
- exercises
- glossary entries
- output comparisons
- mental models

### `session_notes/`

Development notes documenting major design decisions, milestones, and working sessions.
