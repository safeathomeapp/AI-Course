# Module 1.10 Foundations Portfolio Project

## Module Overview

**Tier:** Foundations  
**Estimated Duration:** 120-180 minutes  
**Prerequisites:** Module 1.1 to Module 1.9

### Learning Objectives

By the end of this module, learners will be able to:

- integrate core Tier 1 skills into one coherent project
- produce portfolio artefacts demonstrating practical AI capability
- apply human-review controls across an end-to-end AI-assisted task

---

## 1. Concept Explanation

This project consolidates Tier 1 from isolated techniques into a repeatable practice.

Why this matters:
- employers value demonstrated outputs, not theory alone
- integration reveals workflow gaps earlier
- clear artefacts support progression into applied and capstone work

How AI behavior relates to this concept:
- AI can accelerate drafting and synthesis across project stages
- AI cannot replace responsibility for accuracy and quality
- review checkpoints remain essential at every stage

Where learners commonly go wrong:
- over-scoping project complexity
- skipping validation to save time
- producing outputs without reusable structure

Short example: A learner builds a customer-support workflow with structured prompts, triage schema, and review checklist.

---

## 2. Demonstration

### Example Context
Operations and administration: build an AI-assisted process documentation pack.

### Weak Example
"Build a complete AI project that proves I am good at AI."

### Improved Example
"You are a course project assistant.

Context:
I am completing a Tier 1 foundations portfolio project.

Task:
Help me build three deliverables:
1) structured prompt library
2) AI-assisted research brief
3) first workflow blueprint

Constraints:
- Keep scope realistic for one learner
- include validation and human review checkpoints
- reinforce AI as a tool, not an authority

Output Format:
1. Project plan with stages
2. Deliverable checklist
3. Review rubric"

### Analysis
The improved prompt creates a bounded project with clear outputs and controls. It aligns with Tier 1 outcomes and prepares learners for capstone progression.

## 3. Guided Exercise

### Task
Complete your Tier 1 portfolio package using one domain of your choice.

### Instructions
1. Select a domain (finance, software, product, marketing, operations, or support).
2. Produce the three required deliverables with explicit review steps.
3. Submit a short reflection on what worked, failed, and improved.

### Suggested Structure
- Domain:
- Deliverable 1: Structured prompt library
- Deliverable 2: AI-assisted research brief
- Deliverable 3: First workflow blueprint
- Validation method:
- Reflection summary:

### Example Output
Domain: Product management  
Deliverable 1: 10 structured prompts for discovery and prioritization  
Deliverable 2: Interview synthesis brief with open questions  
Deliverable 3: Workflow for weekly feedback-to-roadmap process  
Validation: Peer review plus evidence checks against source notes

## 5. Common Mistakes

- creating artifacts that are too generic to reuse
- no evidence of verification practice
- producing outputs without clear audience or use case
- treating AI-generated text as final deliverable

These mistakes weaken both learning and career value.
