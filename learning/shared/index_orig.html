Module 1 · Foundations

AI is not an authority. It is a tool.

This module introduces the practical mental model for using AI in real work: prompt, context, generation, review, and decision.

Key principle

AI drafts. Humans review. Humans decide.

What this module is not

This is not a hype piece and not a deep machine learning lecture. It is a practical operating model for responsible AI use.

Learning objectives

  • Explain what AI is in practical workplace terms
  • Describe the role of human review in AI-assisted work
  • Recognise the difference between generation and decision-making
  • Use a basic structured interaction pattern with an AI tool

What AI does well

Produces drafts, options, summaries, reformulations, and first-pass structure quickly when given enough context.

What AI does not do

It does not own accountability, understand consequences like a professional, or automatically know what is true, safe, or suitable for the situation.

Demonstration

Weak prompt vs improved prompt

Prompt

Write me a professional email.

Likely result

Generic, vague, and missing the user’s audience, intent, constraints, and desired tone.

Prompt

You are helping me draft a short professional email to a supplier.
Purpose: request a corrected invoice.
Tone: polite and direct.
Constraints: under 120 words, plain English, include a request for confirmation by Friday.

Likely result

More targeted output because the task, audience, tone, and constraints are specified.

What changed?

  • Role and purpose were defined
  • Audience was made explicit
  • Tone and output limits were constrained
  • The request became reviewable

Guided practice

Improve the prompt

Exercise brief

Take the vague prompt below and rewrite it so an AI tool has enough context to produce a useful first draft.

Starting prompt:

“Summarise this meeting.”

Self-check

  • Did you state the task clearly?
  • Did you define the audience?
  • Did you include context or source material assumptions?
  • Did you specify a format for the answer?
  • Did you include any limits or priorities?

Real-world application

Where this shows up at work

Scenario

A manager asks for a quick first draft of a client update, a summary of notes, and a short action list. AI can accelerate the draft, but a human still needs to check tone, accuracy, omissions, and business risk before sending anything.

Key takeaways

AI generates

It produces language and structure based on patterns.

Humans review

Professionals check reliability, suitability, and risk.

Humans decide

Responsibility stays with the user, not the tool.
