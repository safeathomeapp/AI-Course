# Revision Plan — T1M03

## High-priority changes
1. Address `T1M03-01`: broaden demonstration coverage with concise role-variant examples for under-served cohorts (developer, marketing/content, sales/ops, regulated contexts), while keeping current teaching length discipline.
2. Address `T1M03-02`: add explicit calibration guidance for medium-risk and ambiguous tasks, including a default decision rule when risk is uncertain.
3. Address `T1M03-03`: add a lightweight team-operationalization layer (ownership mapping, escalation trigger, review-gate expectation) so managers can implement beyond individual practice.

## Medium-priority changes
1. Address `T1M03-06`: add matrix maintenance guidance (review cadence, update triggers, role ownership for updates).
2. Address `T1M03-05`: include a richer worked matrix sample (more rows and mixed task types, including early-career style tasks) and concise optional-column guidance.
3. Address `T1M03-04` with an optional regulated-industry callout: clarify when sign-off and documentation carry compliance obligations.

## Low-priority changes
1. Add optional facilitation notes for collaborative/team workshop usage of the matrix.
2. Add optional portfolio/deployment communication pointers for learners presenting the artefact upward.
3. Add optional note acknowledging that some contextual factors (culture/cognitive load/time pressure) can increase misallocation risk.

## Rejected suggestions
1. Reorder curriculum so T1M03 precedes T1M02 (`T1M03-08`): rejected for now due low evidence in this set and unresolved downstream sequencing implications.
2. Expand module into extensive business-packaging/ROI playbook (`T1M03-09`): rejected as out-of-scope for this foundation module.
3. Add many niche persona-specific branches in core flow (`T1M03-07`): rejected in core layer; better handled as optional annex/examples.

## Notes on what should not be changed in the module
1. Do not remove or dilute the opening allocation-first thesis.
2. Do not increase technical complexity; current readability is a major strength and clear improvement over T1M02.
3. Do not remove the three-tier risk model or its review-depth mapping.
4. Do not remove the matrix exercise structure, checklist, or reflection prompts.
5. Do not remove the key principle preserving human authority over correctness/risk/final decisions.
6. Do not replace the practical governance framing with abstract AI theory.
