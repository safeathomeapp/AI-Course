# Review Synthesis — T1M02

## Module Under Review
- Module ID: T1M02
- Module title: How Large Language Models Actually Work
- Source module: `modules/foundation/T1M02_how_llms_work.md`

## Review Sample
- Feedback files reviewed: 14/14
- Reviewer profile source: `reviews/reviewer_profiles.md`
- Reviewer types represented:
  - beginners / low-tech-comfort managers
  - experienced and heavy AI users
  - developers / technical practitioners
  - operations and business leaders
  - educators / enterprise L&D evaluators
  - creative professionals
  - non-native English and neurodivergent learners
  - compliance/regulated-industry stakeholders

## Repeated Strengths
1. The process pipeline/mental model was widely praised as a clear structure for understanding LLM behavior.
2. Probability-ranking plus hallucination mechanics produced the strongest “aha” moment across persona types.
3. Context-window explanation was broadly useful and easily transferable to real work constraints.
4. Guided comparison exercise (unconstrained vs uncertainty-aware) repeatedly demonstrated visible reliability improvement.
5. Portfolio artefact (LLM Reliability Control Card) was seen as practical and adaptable to real workflows.
6. Explicit human verification step remained a trusted anchor and aligned with course philosophy.
7. Many reviewers reported immediate behavior changes in prompting and verification after completion.

## Repeated Concerns
1. Early technical density creates intimidation and can block non-technical engagement before practical relevance is felt.
2. Terminology clarity remains weak for mixed audiences (`tokens`, `context window`, `tokenization`).
3. Example set is perceived as role-imbalanced; software demo is highly relevant for some, inaccessible for many others.
4. Exercise needs stronger calibration support (worked example, clearer quality criteria/rubric).
5. Artefact completion still lacks sufficient guided examples for consistent learner confidence.
6. Time-to-complete variance is wide; non-technical learners often exceed stated duration.

## High-Priority Confirmed Issues
1. `T1M02-01` — Early technical load creates repeated confidence/completion risk in non-technical cohorts.
2. `T1M02-02` — Terminology is still under-defined for mixed-audience delivery.
3. `T1M02-03` — Example relevance gap persists across role groups.
4. `T1M02-04` — Exercise scaffolding remains insufficiently calibrated.

## Possible Issues To Watch
1. `T1M02-07` — Regulated-industry governance/compliance framing remains low-frequency but high-severity.
2. `T1M02-06` — Time estimate mismatch may impact completion planning in enterprise cohorts.
3. `T1M02-08` — Tokenization depth may be over-tuned for broad practitioner audiences.

## Outliers / Low-Value Feedback
1. `T1M02-09` advanced deep-dive requests are valid for power users but not broadly representative of foundation-learner needs.
2. `T1M02-10` procurement/deployment infrastructure asks are useful for program operations, not module-core pedagogy.

## Stability Notes
What should remain unchanged based on strong repeated evidence:
1. Keep the pipeline mental model and explicit human verification endpoint.
2. Keep probability/hallucination mechanism explanation as a core teaching pillar.
3. Keep the comparative prompt exercise structure (A vs B prompt contrast).
4. Keep the reliability artefact concept (Control Card) as a practical transfer mechanism.
5. Keep accountability framing aligned with “AI assists, humans decide.”

## Revision Priorities
1. Reduce early friction for non-technical learners without removing essential mechanics.
2. Improve terminology onboarding and practice calibration quality.
3. Broaden perceived role transfer via minimal, targeted demonstration diversification.
4. Preserve high-value conceptual core while adjusting sequencing and support scaffolds.
