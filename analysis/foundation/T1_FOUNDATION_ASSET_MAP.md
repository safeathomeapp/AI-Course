  ### Design Notes

  - Why these assets were selected
      - Mapped each module to its primary instructional bottleneck from Foundation analysis:
        T1M01 onboarding/accountability, T1M02 uncertainty reasoning, T1M03 risk governance, T1M04 prompt structure/control.
      - Each module gets at least one of every required asset type (diagram, worked example, exercise, output comparison, artefact,
        glossary, mental model) without adding unnecessary volume.
  - Where reuse occurs
      - AI Workflow Model and human-review vocabulary are reused across all modules for consistency.
      - Comparison and worked-example assets are reused cross-module where pedagogically adjacent (for example KPI/reporting appears
        in both T1M02 and T1M03 contexts).
      - Exercise library is reused with increasing independence (beginner-safe -> adaptation -> independent).
  - How this supports progressive learning (T1M01 -> T1M04)
      - Sequence moves from safe use model (T1M01) to reliability interpretation (T1M02), then risk-based governance (T1M03), then
        output control via structured prompting (T1M04).
      - The mapping intentionally preserves the course philosophy: AI drafts, humans review, humans decide.
