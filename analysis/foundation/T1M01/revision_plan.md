# T1M01 Revision Plan (Evidence-Based)

## High-priority changes
1. Add beginner-safe exercise entry path.
   - Include explicit fallback for learners with no existing AI prompts.
   - Clarify acceptable prompt sources (work, internship, study, hypothetical task).
   - Add explicit tool-access note for exercise execution.
2. Define key terms at first use or in a compact glossary.
   - Prioritize `context window`, `tokens`, and AI-specific `hallucination` usage.
3. Strengthen exercise/artefact calibration.
   - Provide one full worked exercise example end-to-end.
   - Add simple 1-5 anchor guidance for evaluation criteria.
   - Add risk-level guidance plus one completed checklist row.
4. Expand perceived role transfer with minimal scope increase.
   - Add at least one additional example/callout from a non-current domain cluster (e.g., technical/support/customer-facing).

## Medium-priority changes
1. Tighten or reposition low-yield historical context section.
   - Keep essential orientation, reduce perceived filler.
2. Improve readability structure for higher-friction cohorts.
   - Break dense paragraphs.
   - Reduce idiomatic phrasing where avoidable.
   - Use clearer visual hierarchy (callouts/checkpoints) for critical patterns.
3. Add timing expectation note for learner variability.
   - Preserve 60-90 baseline while acknowledging some learners may need longer.

## Low-priority changes
1. Add optional “stretch” variant for experienced users in the exercise.
2. Add optional role-specific example bank (appendix/supplement) if resources allow.
3. Add explicit note on iterative prompting loop (review-refine-repeat) for workflow realism.

## Rejected suggestions
1. Rebuild T1M01 into role-specific tracks (dev/sales/creative/ops) inside this module.
   - Rejected: exceeds foundation scope and risks fragmentation.
2. Add enterprise procurement/ROI platform details directly to module body.
   - Rejected: belongs to sales/onboarding documentation, not core lesson.
3. Shift module focus from safe operational framing to aggressive capability framing.
   - Rejected: conflicts with course philosophy and repeated reviewer support for current framing.
4. Treat single-reviewer certification/LinkedIn signaling requests as core content requirements.
   - Rejected: low-frequency and outside module learning objective.

## Notes on what should not be changed in the module
1. Do not change the core principle: AI as tool, not authority.
2. Do not remove explicit human review and human accountability emphasis.
3. Do not replace the structured prompt framework fields; they are a major success signal.
4. Do not remove before/after demonstration pedagogy; it is repeatedly effective.
5. Do not remove the operating checklist artefact concept; it is one of the strongest transfer elements.
