# PROJECT_CONTEXT.md

## Purpose of This File

This file exists primarily to orient AI tools and contributors quickly.

Any AI agent, coding assistant, or contributor working in this repository should read this file before making changes.

## Project Identity

This repository is the working source for a three-tier educational course series titled:

**An AI's Guide for Humans: Understanding and Harnessing Artificial Intelligence**

The course is designed to teach learners how to understand AI systems properly, use them deliberately, and integrate them into real work.

## High-Level Philosophy

The course is built on these principles:

- AI is a tool, not an authority.
- Most people use AI badly because they use it casually.
- Prompt tricks are not the real skill.
- The real skill is structured thinking, workflow design, and controlled implementation.
- AI should amplify human capability, not replace human judgment.
- The course must provide practical and career-relevant value.

## Audience Model

The course serves a mixed audience in the early tiers and becomes more advanced in the final tier.

### Tier 1 — Foundations
Broad audience:
- knowledge workers
- career changers
- professionals trying to improve their work
- curious learners with little or no technical depth

### Tier 2 — Applied
More work-focused audience:
- professionals integrating AI into real workflows
- managers, analysts, operators, writers, product staff
- semi-technical learners

### Tier 3 — Systems
More advanced audience:
- builders
- technical professionals
- ambitious operators
- learners interested in designing AI-enabled systems

## Course Structure

The three tiers are independent but connected.

Learners should be able to:
- start at Tier 1 and progress naturally
- skip Tier 1 if they already understand basic AI usage
- take Tier 2 or Tier 3 as stand-alone courses where appropriate

Progression model:
**Use → Integrate → Design**

Program flow for learner outcomes:
1. Portfolio work through modules
2. Reflection checkpoints
3. Choose project category
4. Design personal AI workflow
5. Capstone build

## Non-Negotiable Writing Rules

Any content created in this repository should:

- be direct and useful
- avoid hype and inflated claims
- avoid hand-holding fluff
- prioritize real professional value
- align with the fixed module template
- include tangible exercises and portfolio artefacts
- reinforce the career amplifier narrative
- remain platform-neutral unless a tool-specific example is explicitly required

## Repository Roles

### `docs/`
Governing documents for the course. These are the source of truth.

### `foundation_course/`
Modules and materials for the 20-hour foundations tier.

### `applied_course/`
Modules and materials for the 20–30 hour applied workflows tier.

### `systems_course/`
Modules and materials for the 30–40 hour advanced systems tier.

### `prompts/`
Canonical prompts, examples, and reusable prompt assets.

### `capstones/`
Capstone briefs, marking rubrics, and final project guidance.

## Recurring Teaching Domains

These example domains should be used repeatedly throughout the course:

- Finance and bookkeeping
- Software development
- Product management
- Marketing and content creation
- Operations and administration
- Customer support

Reuse of these domains is intentional. It improves consistency and helps learners see how concepts transfer across contexts.

## Required Module Pattern

Every module should follow the pattern defined in `docs/COURSE_MODULE_TEMPLATE.md`:

1. Module Overview
2. Concept Explanation
3. Demonstration
4. Guided Exercise
5. Real-World Application
6. Common Mistakes
7. Portfolio Artefact
8. Reflection Questions
9. Optional Advanced Exploration

## What Good Output Looks Like

Good output in this repository is:

- clear
- structured
- reusable
- consistent
- realistic
- professionally relevant

Bad output in this repository is:

- generic
- overhyped
- repetitive
- tool-worshipping
- based on shallow prompting tips
- disconnected from real work

## Instruction for AI Tools

Before generating or editing content:

1. Read this file.
2. Read `README.md`.
3. Read the files in `docs/`.
4. Match the tone and structure already established.
5. Do not invent a different philosophy for the course.
6. Do not collapse the course into a beginner-only prompt guide.
7. Do not make the content feel like marketing copy.
8. Confirm that session or milestone notes are captured in `docs/Session Notes/`.
9. If notes were not requested, proactively prompt to add them before closing work.
10. Ensure content supports the portfolio-to-capstone flow defined in `docs/AI_COURSE_ARCHITECTURE.md`.

## Priority Order for Content Decisions

When making writing decisions, prefer:

1. clarity over cleverness
2. structure over improvisation
3. usefulness over novelty
4. transferable principles over tool-specific tricks
5. practical learning value over impressive wording
